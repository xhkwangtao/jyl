# 首页公告与页面统计系统设计

## 1. 背景

九眼楼小程序需要运营人员能够在后台配置首页公告，同时需要统计小程序页面访问情况，用于判断首页、地图、研学报告、支付墙等关键页面的访问量和用户路径。

本模块定位为“运营配置与访问统计”，独立于内容发布、支付、权益和研学业务。公告可以随时上下线，不要求走内容发布流程；页面统计只记录访问行为，不记录用户输入内容、聊天内容、答题卡图片或支付密钥等敏感信息。

## 2. 首页公告

后台运营可配置首页公告：

- 标题
- 内容
- 公告类型：`system`、`activity`、`normal`
- 展示形式：`banner`、`modal`、`list`
- 跳转类型：`none`、`miniprogram`、`webview`
- 跳转地址
- 排序权重
- 启用状态
- 生效时间
- 失效时间
- 结构化内容块 `content_blocks`

小程序接口：

```text
GET /api/v1/client/announcements/home
```

返回规则：

- 只返回 `enabled=true` 的公告。
- 当前时间必须在 `starts_at` 和 `ends_at` 范围内。
- `starts_at` 为空表示立即生效。
- `ends_at` 为空表示长期有效。
- 按 `priority` 从高到低排序。
- 如果存在 `content_blocks`，小程序优先按内容块渲染；如果没有，则回退展示 `content` 纯文本。

### 2.1 内容块协议

公告不直接下发 HTML，而是下发小程序可渲染的结构化内容块。

支持的块类型：

| 类型 | 说明 |
|------|------|
| `image` | 图片块，支持图片比例、填充方式和点击链接 |
| `heading` | 标题块，支持整块点击 |
| `paragraph` | 正文块，支持整块点击 |
| `button` | 按钮块，支持点击跳转 |

块级链接结构：

```json
{
  "type": "miniprogram",
  "url": "/pages/routes/index",
  "params": {
    "route_id": 3
  }
}
```

第一阶段支持的链接类型：

| 类型 | 说明 |
|------|------|
| `none` | 不跳转 |
| `miniprogram` | 跳转小程序内部页面 |
| `webview` | 跳转小程序 web-view 页面 |

完整示例：

```json
{
  "id": 1,
  "title": "五一假期推荐路线",
  "content": "建议错峰游览，提前查看推荐路线。",
  "display_type": "modal",
  "content_blocks": [
    {
      "id": "block_1",
      "type": "image",
      "asset_id": 123,
      "url": "https://cdn.example.com/notice.jpg",
      "alt": "五一活动",
      "aspect_ratio": "16:9",
      "fit": "cover",
      "link": {
        "type": "miniprogram",
        "url": "/pages/routes/index",
        "params": {
          "route_id": 3
        }
      }
    },
    {
      "id": "block_2",
      "type": "heading",
      "text": "五一假期推荐路线",
      "level": 2,
      "link": {
        "type": "none"
      }
    },
    {
      "id": "block_3",
      "type": "paragraph",
      "text": "建议错峰游览，提前查看推荐路线。",
      "link": {
        "type": "miniprogram",
        "url": "/pages/routes/index"
      }
    },
    {
      "id": "block_4",
      "type": "button",
      "text": "查看路线",
      "style": "primary",
      "link": {
        "type": "miniprogram",
        "url": "/pages/routes/index"
      }
    }
  ]
}
```

后台公告页面直接复用资源上传接口：

```text
POST /api/v1/admin/assets/upload
```

上传成功后，公告图片块保存：

- `asset_id`
- `url`
- `alt`
- `aspect_ratio`
- `fit`
- `link`

### 2.2 小程序使用方法

小程序首页加载公告：

```js
async function fetchHomeAnnouncements() {
  const res = await request({
    url: '/api/v1/client/announcements/home',
    method: 'GET'
  })
  return res.data.items || []
}
```

统一处理公告链接：

```js
function buildQuery(params = {}) {
  const query = Object.keys(params)
    .map((key) => `${key}=${encodeURIComponent(params[key])}`)
    .join('&')
  return query ? `?${query}` : ''
}

function handleAnnouncementLink(link) {
  if (!link || link.type === 'none') {
    return
  }

  if (link.type === 'miniprogram') {
    wx.navigateTo({
      url: `${link.url}${buildQuery(link.params)}`
    })
    return
  }

  if (link.type === 'webview') {
    wx.navigateTo({
      url: `/pages/webview/index?url=${encodeURIComponent(link.url)}`
    })
  }
}
```

渲染规则：

```js
function normalizeAnnouncementBlocks(announcement) {
  if (announcement.content_blocks && announcement.content_blocks.length) {
    return announcement.content_blocks
  }
  return [
    {
      id: 'fallback_content',
      type: 'paragraph',
      text: announcement.content,
      link: { type: 'none' }
    }
  ]
}
```

小程序组件建议：

```text
AnnouncementRenderer
```

组件输入：

```js
{
  announcement,
  onLinkTap: handleAnnouncementLink
}
```

渲染映射：

| 内容块 | 小程序组件 |
|--------|------------|
| `image` | `<image mode="aspectFill/aspectFit">` |
| `heading` | `<view class="announcement-heading">` |
| `paragraph` | `<view class="announcement-paragraph">` |
| `button` | `<button>` |

图片块渲染要点：

- `fit=cover` 对应 `mode="aspectFill"`。
- `fit=contain` 对应 `mode="aspectFit"`。
- 点击图片时调用 `handleAnnouncementLink(block.link)`。
- `aspect_ratio` 由样式控制，例如 `16:9`、`4:3`、`1:1`。

后台接口：

```text
GET    /api/v1/admin/announcements
POST   /api/v1/admin/announcements
PATCH  /api/v1/admin/announcements/{announcement_id}
DELETE /api/v1/admin/announcements/{announcement_id}
```

## 3. 页面统计

小程序进入关键页面时调用：

```text
POST /api/v1/client/analytics/page-view
```

请求示例：

```json
{
  "page_path": "/pages/home/index",
  "page_title": "首页",
  "scene": "launch",
  "source": "tabbar",
  "referrer": "",
  "anonymous_id": "local-device-id",
  "platform": "ios",
  "wechat_version": "8.0.5",
  "device_model": "iPhone",
  "network_type": "wifi",
  "extra": {
    "poi_id": 12
  }
}
```

鉴权规则：

- 该接口不强制登录。
- 如果请求带有效 JWT，后端记录 `client_user_id` 和 `openid`。
- 如果未登录，后端使用 `anonymous_id` 辅助计算 UV。
- JWT 无效时不阻断统计，只按匿名访问记录。

IP 规则：

- 小程序不传 IP。
- 后端从 `X-Forwarded-For`、`X-Real-IP`、`request.client.host` 依次解析真实 IP。
- `X-Forwarded-For` 只取第一个 IP 作为 `client_ip`，原始值保存到 `raw_forwarded_for`。
- 后端使用纯真 IP 数据库解析 IP 归属。

纯真库位置：

```text
backend/data/ipdb/qqwry.dat
```

配置项：

```text
APP_QQWRY_DATABASE_PATH
```

默认值指向项目内 `backend/data/ipdb/qqwry.dat`。

## 4. 后台统计

后台接口：

```text
GET /api/v1/admin/analytics/overview
GET /api/v1/admin/analytics/pages
GET /api/v1/admin/analytics/page-views
```

后台页面：

```text
/announcements  首页公告
/analytics      页面统计
```

页面统计展示：

- 总 PV
- 总 UV
- 今日 PV
- 今日 UV
- 页面排行
- 最近访问明细
- IP
- IP 归属地
- 平台、设备、微信版本
- 来源和上级页面

UV 计算规则：

```text
优先 client_user_id
其次 anonymous_id
最后 client_ip
```

## 6. 隐私与合规边界

允许记录：

- 页面路径
- 页面标题
- 访问来源
- 登录用户 ID
- 匿名设备 ID
- IP 和 IP 归属地
- 设备、平台、微信版本

不记录：

- 用户聊天内容
- 答题卡图片内容
- 手机号
- 微信头像
- 支付密钥
- 微信支付回调完整密文
